model: aquí irán los modelos, para ser fieles al paradigma orientado objetos tenemos que tener 
una clase por cada tabla o entidad de la base de datos(excepto para las tablas pivote) y estas 
clases servirán para crear objetos de ese tipo de entidad(por ejemplo crear un objeto usuario 
para crear un usuario en la BD). También tendremos modelos de consulta a la BD que contendrán 
consultas más complejas que estén relacionadas con una o varias entidades.
